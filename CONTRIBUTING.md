## 🌟 Why Contribute?

By contributing to this hub, you are:

- 🌍 **Helping developers worldwide** discover resources they might never have found
- 🧠 **Sharing your expertise** with a growing global community
- 🏆 **Building your open source portfolio** and GitHub activity
- 🤝 **Giving back** to the community that helped you grow
- ⭐ **Getting credited** in a repository used by thousands of developers

---

## 📜 Code of Conduct

We are committed to providing a **welcoming, inclusive, and harassment-free** experience for everyone.

By participating, you agree to:

- ✅ Use welcoming and inclusive language
- ✅ Respect differing viewpoints and experiences
- ✅ Gracefully accept constructive criticism
- ✅ Focus on what is best for the community
- ❌ Not use sexualized language or imagery
- ❌ Not troll, insult, or make personal attacks
- ❌ Not publish others' private information without permission

Violations may result in contributions being declined and access being removed.

## 🚀 Quick Start

```bash
# 1. Fork the repository on GitHub
#    (Click the Fork button at the top right of the repo page)

# 2. Clone YOUR fork to your local machine
git clone https://github.com/augustdev290/the-developer-universe-hub.git
cd the-developer-universe-hub

# 3. Create a new branch for your contribution
git checkout -b add/your-resource-name

# 4. Make your changes to the appropriate file in /resources/

# 5. Stage and commit your changes
git add .
git commit -m "add: [Collection Name] Resource Title — one-line description"

# 6. Push your branch to your fork
git push origin add/your-resource-name

# 7. Open a Pull Request on GitHub
#    Go to the original repo and click "Compare & pull request"
```

## 🚫 What NOT to Submit

To keep the hub high-quality, we **will not accept**:

- ❌ Resources that are broken, abandoned, or no longer maintained
- ❌ Duplicate resources already listed in the collection
- ❌ Clickbait, spam, SEO-farm, or low-quality content
- ❌ Resources that promote harmful, unethical, or illegal activities
- ❌ Personal portfolio projects without significant community value
- ❌ Resources behind excessive paywalls without mentioning cost
- ❌ Affiliate or referral links that manipulate the URL for commission
- ❌ Self-promotion disguised as a neutral resource (must be genuinely useful)

---

## ✅ Quality Standards

Every resource must meet **all** of the following:

| Standard | Requirement |
|:---------|:------------|
| 🟢 **Active** | The resource is actively maintained or is a timeless classic |
| 🟢 **Accessible** | Free resources preferred; paid resources must be clearly labeled |
| 🟢 **Relevant** | Directly useful to developers in the section's topic |
| 🟢 **Trustworthy** | From a reputable author, organization, or institution |
| 🟢 **English** | Primary language should be English (other languages are bonus) |
| 🟢 **Unique** | Not already listed elsewhere in the same collection |

## 🖊️ Formatting Guide

Please follow the table format already used in each resource file. Here is the standard:

### Adding a Resource to an Existing Table

```markdown
| [📖 Resource Name](https://example.com) | 📋 Type | 🟢 Beginner | 🆓 Free |
```

**Level icons:**
- 🟢 = Beginner
- 🟡 = Intermediate
- 🔴 = Advanced
- 🟢🟡🔴 = All Levels

**Cost icons:**
- 🆓 = Free
- 💰 = Paid
- 🆓/💰 = Freemium (free tier + paid plan)
- 💰 Audit Free = Paid course with free audit option

### Commit Message Format

```
add: [Collection] Resource Title — one-line description
fix: [Collection] Update broken link for Resource Name
update: [Collection] Improve description for Resource Name
remove: [Collection] Resource Name — reason (e.g., link dead)
docs: Improve section header for Topic Name
```

## 🔄 Pull Request Process

1. **Ensure your PR is focused** — one topic or a small batch of related resources per PR
2. **Fill out the PR template** completely when prompted
3. **Check for duplicates** before submitting — search the file for the resource name
4. **Verify all links** work before submitting
5. **Follow the formatting guide** exactly — inconsistent formatting will delay review
6. **Be patient** — we review PRs within **48 hours** on weekdays

### PR Title Format

```
add: [Software Engineer] Resource Name
fix: [Web Dev] 3 broken links in APIs section
update: [Full-Stack] Improve descriptions in Backend section
```

## 🐛 Reporting Issues

Found a broken link, outdated resource, or formatting problem? Please open an issue!

**When reporting, please include:**

- 📋 The resource file and section where the issue is located
- 🔗 The specific resource name and URL (if applicable)
- 📝 A clear description of the problem
- 💡 A suggested fix (optional but appreciated)
